XSS & Open Redirect Chain: Major Streaming Platform

Discovery Date: October 9, 2025

Disclosure Date: October 9, 2025

Vendor Status: Patched

CVSS Score: 8.2 (Critical)

Executive Summary

During security testing of a major streaming platform's subscription flow, I discovered a critical vulnerability chain combining reflected XSS with open redirects, allowing complete account takeover and phishing attacks against millions of users.

Technical Impact

  • Complete account takeover through session cookie theft
  • Phishing attacks abusing trusted domain reputation
  • Mobile app hijacking through deep link manipulation
  • Device feature abuse (phone calls, SMS, messaging apps)

Proof of Concept

Example XSS Payload:

# JavaScript execution in user context
https://platform.com/subscriptionconfirm?returnTo=javascript:alert(document.domain)

# Reflected in meta tags:
<meta property="app:ios:url" content="app://confirm?returnTo=javascript%3Aalert(document.domain)"/>

Remediation

  • Implement strict input validation and URL sanitization
  • Deploy Content Security Policy to block inline JavaScript
  • Whitelist allowed URL schemes and domains
  • Conduct comprehensive security testing of all user-input parameters

Business Impact

Demonstrated complete account compromise capabilities against a major streaming platform with millions of users, highlighting critical gaps in input validation and client-side security controls.
